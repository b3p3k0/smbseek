## Documentation Map

- **Guides (`guides/`)** – user-facing workflows and references  
  - `USER_GUIDE.md` – CLI quickstart and workflows  
  - `XSMBSEEK_USER_GUIDE.md` – GUI usage, including read-only file browser  
  - `EXTRACT_WORKFLOW_GUIDE.md` – controlled extraction/quarantine pipeline  
  - `ENHANCED_SHARE_TRACKING_USER_GUIDE.md` – share tracking UX  
  - `RCE_SIGNATURE_GUIDE.md`, `RANSOMWARE_INDICATORS.md` – detection content  

- **Operations (`operations/`)** – migrations and runbooks  
  - `DATABASE_MIGRATION_GUIDE.md`, `xsmbseek_migration.md`

- **Development (`dev/`)** – contributor docs, integration, roadmap  
  - `AI_AGENT_FIELD_GUIDE.md`, `COLLAB.md`, `DOC_GUIDE.md`  
  - `GUI_BACKEND_UPDATES.md`, `GUI_INTEGRATION_GUIDE.md`, `FRONTEND_INTEGRATION_RECOMMENDATIONS.md`  
  - `XSMBSEEK_DEVNOTES.md`, `FUTURE.md`, `RCE_SIGNATURE_WISHLIST.md`

- **Release (`release/`)**  
  - `XSMBSEEK_CHANGELOG.md`

- **Archive (`archive/`)**  
  - Deprecated references (e.g., `SANDBOXED_EXPLORER_GUIDE.md`)

When adding new docs, place them in the closest-fit folder above and keep user-facing guides concise, linking to deeper dev notes when needed.
